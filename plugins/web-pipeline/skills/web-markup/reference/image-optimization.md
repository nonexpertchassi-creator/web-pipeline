# 이미지 최적화 (webp · SEO 파일명 · 설명형 alt)

피그마 등에서 가져온 이미지는 그대로 넣지 않는다. 퍼블리셔가 자주 누락하는 세 가지를 처리한다: **무겁다(webp 변환)**, **파일명이 의미 없다(SEO 이름)**, **alt가 빈약하다(설명형)**.

## 1. webp 변환 (1MB↑)

1MB 넘는 `png`/`jpg`는 webp로 바꾼다(품질 ~80). 보통 용량이 크게 준다. webp는 최신 브라우저 전부 지원하므로 `<picture>` 폴백은 대개 불필요하다(아주 오래된 브라우저까지 지원해야 할 때만).

```bash
bash ${CLAUDE_SKILL_DIR}/scripts/optimize-images.sh <대상폴더>   # 예: ... ./assets
# cwebp / ImageMagick / Pillow 중 있는 도구로 변환, 원본은 보존
```

> 스크립트는 자동 실행이 아니라 **Claude가 셸로 실행**한다. 경로는 `${CLAUDE_SKILL_DIR}/...`(스킬 설치 위치 무관). 셸 환경 + 변환 도구(cwebp/ImageMagick/Pillow 중 하나)가 있어야 동작한다 — 없으면 스크립트가 에러로 알려준다.

- 변환 후 마크업·CSS의 참조를 `.webp`로 바꾼다(아래 파일명 단계에서 함께 처리하면 한 번에).
- 참조를 다 바꾸고 화면 확인이 끝나면 원본(png/jpg)을 정리한다. (서빙되는 건 webp만)
- 치수가 과도하게 큰 원본은 표시 크기에 맞게 줄여 내보내는 것도 검토(불필요한 해상도는 용량 낭비).

## 2. SEO 친화 파일명

`IMG_2931.png`·`download(3).jpg` 같은 이름은 검색(특히 이미지 검색)에 무의미하다. **브랜드 + 내용** 기반의 의미 있는 이름으로 바꾼다.

- 형식: 소문자·하이픈(kebab-case), 영문/숫자. 예: `hanmac-extra-can-smoooth.webp`, `wheelchair-patient-illustration.webp`.
- **이름은 Claude가 이미지를 직접 보고(비전) 제안한다.** 단 **브랜드·제품명**은 보이는 것만으로 모를 수 있으니 `CLAUDE.md`의 용어집을 참고하고, 없으면 사용자에게 묻는다. (예: 캔의 "smoooth"는 읽지만 그게 "한맥 엑스트라"인지는 알려줘야 정확)
- 이름을 바꾸면 **참조를 함께 갱신**한다:

```bash
bash ${CLAUDE_SKILL_DIR}/scripts/rename-image.sh <기존경로> <새파일명>
# 예: bash ${CLAUDE_SKILL_DIR}/scripts/rename-image.sh assets/IMG_2931.webp hanmac-extra-can-smoooth.webp
# 파일 이동(git이면 git mv) + 코드 내 모든 참조 자동 교체
```

- ⚠ **코드 밖 참조는 자동으로 못 잡는다** — CMS·DB·SEO 메타 관리자·외부 링크에 박힌 경로는 직접 확인한다. (이 위험 때문에 바꾼 목록을 핸드오프에 남긴다)

## 3. 설명형 alt (보이지 않는 사람에게 설명하듯)

alt는 키워드 나열도, 파일명 복붙도 아니다. **이미지가 전하는 의미를, 보지 못하는 사람에게 한두 문장으로 설명하듯** 쓴다.

판단 기준:

- **장식용**(순수 꾸밈, 의미 없음) → `alt=""`. 스크린리더가 건너뛴다.
- **정보성**(일러스트·사진·제품컷) → 장면·대상을 묘사한다. 무엇이 보이는지, 맥락상 왜 거기 있는지.
- **기능성**(아이콘 버튼·로고 링크) → 모양이 아니라 *기능/목적*을. 예: 돋보기 아이콘 버튼 → `alt="검색"`.
- **복잡**(차트·인포그래픽) → 핵심 데이터/결론을 문장으로. 너무 길면 본문이나 별도 설명에 풀고 alt는 짧게.

톤·길이:

- 보통 한두 문장 이내, 핵심만. 장황하지 않게.
- 브랜드·제품명은 `CLAUDE.md` 용어로 정확히 넣는다.
- 참고: 스크린리더가 이미 "이미지"라고 읽어주므로 "~이미지가 있습니다"는 군더더기일 때가 많다 — 빼고 바로 묘사해도 된다. (스타일은 유연하게, 의미 전달이 우선)

좋은 예:

- 일러스트 → `휠체어를 탄 환자를 그린 일러스트`
- 제품컷 → `한맥 엑스트라 캔에 황금빛 'smoooth' 슬로건이 적혀 있다`
- 차트 → `2024년 분기별 매출 추이 — 4분기에 전 분기 대비 30% 증가`

피해야 할 예:

- `이미지` / `사진1` / `img_2931` — 아무 정보 없음.
- `한맥, 맥주, 캔, smoooth, 광고, 황금색` — 키워드 나열(검색 스팸처럼 읽힘).
- 장식용인데 굳이 긴 설명 — 오히려 스크린리더 사용자에게 소음.

## 핸드오프에 남길 것

- 변환한 이미지(원본→webp) 목록
- 바꾼 파일명(기존→새이름)과 **코드 밖에서 직접 확인이 필요한 참조** 표시
- 복잡 이미지의 긴 설명을 본문/별도에 둔 경우 그 위치
